# Investment Banking

Skill **guarda-chuva** de IB: dá o contexto (o que bancos fazem, processo, capital markets), a **base técnica canônica**, e **roteia** para as skills profundas (valuation, financial-modeling, mna, lbo, corporate-finance) conforme a pergunta. Também serve de base de conhecimento e preparação de entrevista.

## Quando usar
- Visão geral de IB: divisões, coverage vs product groups, o que cada um faz, fees.
- Processo de **deal** (sell-side/buy-side) e **documentos** (pitchbook, teaser, CIM, fairness opinion).
- **IPO / capital markets** (ECM e DCM): como uma empresa levanta capital.
- Revisar **technicals** de IB (accounting, EV vs equity, valuation/DCF/merger/LBO em resumo).
- **Preparação de entrevista** (fit + technical) — lateral, PE, ou estudo.
- **Valuation por setor** — como banco/FIG, REIT, óleo & gás, utilities e tech valuam diferente (ver `references/sector-valuation.md`).
- **Roteamento:** quando a pergunta é ampla, identificar e apontar a skill profunda certa.

Esta skill dá o resumo e o contexto; **mande para a skill profunda** quando o usuário quiser construir/calcular de fato.

## Princípios
1. **Conceito antes do número** — explique o porquê, depois a fórmula.
2. **EV vs Equity e as 3 demonstrações ligadas** são a fundação de quase tudo.
3. **Triangule valuation** (nunca uma metodologia só).
4. **Roteie:** identifique se a pergunta é overview (fica aqui) ou execução (vai para a skill profunda).
5. **Honestidade técnica:** premissas explícitas, faixas, e mostrar o raciocínio.

## Roteiro
1. **Enquadrar:** a pergunta é overview/processo/technical/entrevista — ou é execução (modelar/calcular)?
2. **Overview/processo:** use `references/overview-and-groups.md` e `references/ipo-and-capital-markets.md`.
3. **Technical:** use `references/technical-qa.md` (e aponte a skill profunda para o detalhe).
4. **Entrevista:** use `references/interview-and-fit.md`.
5. **Execução:** roteie para a skill profunda correta (tabela acima).

## Formato da saída
- **Overview:** estrutura clara (divisões, processo, players), conectando ao quadro geral.
- **Technical:** conceito → mecânica → número, com premissas; aponte a skill profunda para aprofundar.
- **Entrevista:** resposta estruturada (situação→ação→resultado) ou conceito técnico explicado do zero.
